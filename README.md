## вң… WHY MARK MY MIND?

> [!NOTE]

> - **вң…рҹ”Ғ SYNC SCREENS:** Edit on 2 screens (*Mindmap + Markdown*) in real time.

> - **вң…рҹ«і DRAG & DROP:** Reposition any block of text wherever you want.
> - **вң…рҹӨ© 100% Markdown:** The philosophy is to be 100% integrated with your Markdown notes, meaning it gives you absolute control over your notes since you will never need to migrate or convert your notes into other third-party formats.
> - **вң…рҹҺЁ Customizations:**
>   - **Layout & Colors:** Choose the layout direction and your color scheme flow. рҹҢҲ
>   - **Line Form:** Choose the line style (*Curved/Rounded/Straight*) + thickness. вһ–
>   - **Block Settings:** Control font size, block width, and maximum height.
> 
> **рҹҳҚ FREE:** 100% Free from an Obsidian lover to all other lovers around the world! рҹҢҸ

## рҹ“қ YAML - CONTROLLING MINDMAP VISUALIZATION VIA NOTE

> [!TIP]
> If you want to manually control the visualization of your mindmap, you can use the YAML property codes at the top of your note:
> **Exemple:**
> 
> ```yaml
> ---
> mmm-type: mindmap
> mmm-layout: <>
> mmm-notename: MY MAIN BLOCK NAME EXEMPLE
> ---
> ```

| Available Options | Available Values | What it does |
| :--- | :--- | :--- |
| `mmm-type:` | `mindmap` | This activates the Mindmap on the note *(Note: Leave the note and come back to refresh and make it work)* |
| `mmm-layout:` | `<>`<br>`>`<br>`V`<br>`A` | `<>` Bidirectional Layout<br>`>` Right-side Layout<br>`V` Bottom Layout<br>`A` Top Layout |
| `mmm-notename` | `ExempleName` | When the "Multiple main blocks" option is enabled, a main node block is created to be the parent of all blocks. By default, it takes your note's name, but you can also rename it using this option. |
